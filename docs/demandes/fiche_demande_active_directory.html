Projet : SOC – SIEM Wazuh Demandeur : Corentin Date : 26 juin 2026 Priorité : Haute

1. Contexte

Je déploie un SIEM Wazuh pour superviser la sécurité du parc informatique de l'école. Le Manager est opérationnel sur ProxFibre. L'étape suivante consiste à installer automatiquement l'agent sur les postes Windows du domaine via une stratégie de groupe (GPO).

L'agent Wazuh collecte uniquement les logs de sécurité Windows (EventLog). Il ne modifie aucun fichier utilisateur, n'intercepte pas le trafic réseau et ne ralentit pas les postes.

2. Ce dont j'ai besoin

2.1 Unité d'Organisation (OU) dédiée

ÉlémentDétail
Nom de l'OUOU=SOC_Monitored,OU=Computers,DC=school,DC=local (à adapter)
ContenuLes comptes ordinateur des postes à superviser
PourquoiIsoler le périmètre de déploiement (déploiement progressif possible)
Alternative : Un groupe de sécurité (GRP_SOC_Monitored) filtré via « Security Filtering » sur la GPO convient aussi.

2.2 Compte de service

ÉlémentDétail
Nomsvc_wazuh_deploy
TypeCompte de service managé (gMSA) ou compte utilisateur standard
Mot de passeNe doit jamais expirer
GroupesDomain Users uniquement (aucun droit admin)
UsageAuthentification API Wazuh (port 55000) pour enregistrer les postes
Sécurité : Le mot de passe est chiffré localement via DPAPI (Data Protection API). Il n'est jamais stocké en clair.

3. Transparence : ce que fait le script

PhaseActionImpact
1Vérifie le tunnel réseauAucun (lecture seule)
2Teste la connectivité APIAucun (test TCP)
3Déchiffre le credential DPAPIAucun (lecture mémoire)
4S'authentifie auprès de l'API WazuhAucun (appel HTTPS sortant)
5Enregistre le posteAucun (appel API)
6Copie le MSI depuis le partage≈ 9 Mo dans C:\Wazuh_Deploy\
7Installation silencieuseService WazuhSvc installé
8Démarrage du serviceService actif (démarrage automatique)
Idempotence : Si l'agent est déjà installé, le script ne fait rien et se termine en < 1 seconde.
